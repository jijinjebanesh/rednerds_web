================================================================================
OVERVIEW
================================================================================

The Dashboard is the central hub for monitoring manufacturing quality, testing 
metrics, debugging operations, and repair management. It provides real-time 
analytics with AI-powered insights into product quality, testing trends, and 
operational performance.

The dashboard is designed to give stakeholders a quick, comprehensive view of:
- Product quality and testing performance
- Debugging and troubleshooting activities
- Manufacturing metrics and repair cases
- Issue identification and root cause analysis
- Testing trends and product status

================================================================================
KEY FEATURES & SECTIONS
================================================================================

1. MASTER KPI CARDS (Top Section)
===============================

Four prominent metric cards with gradient backgrounds display key performance 
indicators:

   a) QUALITY SCORE
      - Shows the overall pass rate percentage (e.g., 78.5%)
      - Displayed as a large formatted number with a colored progress bar
      - Visual indicator: Green progress bar
      - Icon: CheckCircle icon
      - Updates based on testing statistics

   b) DEBUG SESSIONS
      - Total number of debugging sessions initiated/troubleshoot attempts
      - Shows cumulative count of all debug activities
      - Icon: 🔧 Wrench emoji
      - Updates from analytics data

   c) AVERAGE DEBUG TIME
      - Shows average duration per debug session in minutes
      - Example: "45.3m" means average 45.3 minute debug sessions
      - Icon: Speed icon
      - Helps identify if debugging is taking too long

   d) PRODUCTS UNDER TEST
      - Total number of products currently under testing
      - Icon: TrendingUp icon
      - Shows system workload

Each card features:
- Smooth loading states with skeleton loaders
- Color-coded gradient backgrounds for visual appeal
- Formatted numbers (e.g., "1,234" instead of "1234")
- Easy-to-scan layout

2. TABBED ANALYTICS SECTION
===========================

Four tabs allow navigation between different analytical views:

   TAB 1: 🔍 PROBLEM ANALYSIS
   -------------------------
   Displays common problems and issues affecting products

   Shows:
   - Problem Description: Text of the issue encountered
   - Severity Level: Color-coded chips (Critical, High, Medium, Low)
   - Occurrence Count: How many times this problem appeared
   - Affected Products: Number of unique products experiencing the issue
   - Resolution Rate: Percentage showing how often this problem was resolved
   - Visual: Color-coded card backgrounds based on severity
     * Critical: Red background (#d32f2f)
     * High: Orange background (#f57c00)
     * Medium: Yellow background (#fbc02d)
     * Low: Green background (#388e3c)

   Cards display:
   - Problem title (truncated if > 100 characters)
   - Severity badge in the top-right
   - Two stat boxes: "Occurrences" and "Products Affected"
   - Colored progress bar showing resolution success rate


   TAB 2: 📊 TESTING TRENDS
   ----------------------
   Provides visualization of testing metrics over time

   Shows:
   - Test Results Trend Chart (Last 14 Days)
     * Stacked bar chart showing Pass/Partial/Fail results
     * X-axis: Dates formatted as "Mon 1", "Tue 2", etc.
     * Y-axis: Count of tests
     * Color coding:
       - Green bars: Passed tests
       - Orange bars: Partial tests
       - Red bars: Failed tests
     * Interactive tooltips on hover

   - Products by Stage Pie Chart
     * Distribution of products across different manufacturing stages
     * Shows what percentage of products are at each stage
     * Products can be at stages like: Assembly, Testing, QA, Shipped, etc.
     * Color-coded slices with legend


   TAB 3: ⚙️ ROOT CAUSES
   --------------------
   Deep-dive analysis into underlying causes of failures

   Each root cause entry shows:
   - Root Cause Description: Text explaining the technical issue
   - Frequency Count: How many times this root cause was identified
   - Resolution Success Rate: Percentage of fixes that worked
   - Visual indicator: Blue top border, hover effect for interactivity
   - Progress bar showing success rate with color coding:
     * Green (>70%): High success rate
     * Orange (50-70%): Medium success rate  
     * Red (<50%): Low success rate


   TAB 4: 🏭 PRODUCTS STATUS
   -------------------------
   Per-product testing insights and status

   For each product:
   - Product Name: Display name of the product
   - Product ID: Unique identifier
   - Status Badge: "RETEST ⏱️" (warning) or "Ready ✓" (success)
   
   Displays metrics in mini stat boxes:
   - Passed: Number of passed tests (blue background)
   - Failed: Number of failed tests (red background)
   - Debug: Count of debug sessions for this product (purple background)
   - Pass Rate: Percentage of tests that passed (green/orange/red based on rate)
   - Status: Shows "⚠️ Review" if needs retest, "✅ Ready" if all clear

   Visual Indicators:
   - Products needing retest have orange border and light background
   - Cards highlight with shadow on hover
   - Pass rate color coding:
     * Green: >70% pass rate (good)
     * Orange: 50-70% pass rate (caution)
     * Red: <50% pass rate (attention needed)

3. OPERATIONAL METRICS SECTION
===============================

Four legacy metric cards that show crucial operational data:

   a) PRODUCTS MANUFACTURED (This Month)
      - Count of new products made in current month
      - Filters products by manufacturing date
      - Shows only products from current month

   b) FIRST-PASS YIELD
      - Percentage of products that pass testing on first attempt
      - Formula: (Pass tests / Total tests) * 100
      - Example: "87.5%" means most products pass without issues

   c) OPEN DEBUG QUEUE
      - Number of failed/partial tests awaiting debug work
      - Filters tests that don't have resolved debug sessions
      - Indicates workload for debugging team

   d) OPEN REPAIR CASES
      - Count of customer repair cases with status: "received" or "in_progress"
      - Shows how many repairs are still active
      - Excludes completed repairs

   Each card has:
   - Title describing the metric
   - Current value (with retry link if error occurs)
   - Error state with retry button
   - Loading skeleton while fetching

4. ISSUE INSIGHTS SECTION
=========================

Two side-by-side panels:

   LEFT PANEL: Top Recurring Issues (Fuzzy Grouped)
   -----------------------------------------------
   Uses intelligent fuzzy matching to group similar issues together

   Shows:
   - Ranked list of top 6 most common issue clusters
   - Each item numbered (#1, #2, #3, etc.)
   - Cluster description (first 80 chars shown, truncated if longer)
   - Count of issues in this cluster
   - Up to 3 example issues from the cluster
   
   Clustering Algorithm:
   - Normalizes issue text (lowercase, removes special chars, etc.)
   - Calculates Jaccard similarity between issues (token-based)
   - Groups issues with >= 35% similarity together
   - Helps identify patterns even if text is slightly different

   RIGHT PANEL: Symptom Keywords (Testing)
   ----------------------------------------
   Text analysis of test symptoms/failures

   Shows:
   - Top 8 most frequently occurring symptom keywords
   - Keywords are 3+ characters long
   - Listed with occurrence frequency
   - Sorted by how often they appear
   - Helps identify common failure patterns


================================================================================
DATA SOURCES & INTEGRATION
================================================================================

The dashboard pulls data from multiple services:

1. analyticsService.getDashboardAnalytics()
   - Returns: commonProblems, rootCauses, testingStatistics, productInsights
   - Frequency: Loaded on page mount

2. productService.getProducts()
   - Returns: All products with manufacturing dates and stages
   - Used for: Monthly product count, stage distribution

3. testLogService.getTestLogs()
   - Returns: All test results with dates and outcomes
   - Used for: Yield calculation, test trends, symptom analysis

4. debugSessionService.getDebugSessions()
   - Returns: Debug sessions with resolution status
   - Used for: Debug queue size, debug metrics

5. customerRepairService.getRepairs()
   - Returns: Repair cases with status
   - Used for: Open repair case count

================================================================================
DATA LOADING & STATE MANAGEMENT
================================================================================

Initial Load:
- Dashboard fetches all data in parallel when component mounts
- Uses Promise.all() for efficient concurrent requests
- Sets chartLoading = true while data is being fetched

State Variables:
- productsMetric: Monthly product count metric
- yieldMetric: Pass rate percentage
- debugQueueMetric: Pending debug work count
- repairsMetric: Open repair cases count
- commonProblems: List of problem entries
- rootCauses: List of root cause entries
- testingStats: Aggregated testing statistics
- productInsights: Per-product testing data
- issueInsights: Grouped recurring issues
- issueInsightsError: Error message if issue loading fails

Each metric state contains:
- value: Current displayed value
- loading: Boolean indicating if data is being fetched
- error: Error message if load failed

================================================================================
ERROR HANDLING
================================================================================

The dashboard has graceful error handling:

1. Failed Metric Cards
   - Show "Failed to load" message
   - Display clickable "Retry" link
   - Allow user to retry individual metrics

2. Chart Loading Errors
   - Display skeleton loaders while loading
   - Show "No data available" messages if empty
   - Show error text if fetching failed

3. Retry Mechanism
   - Each metric has individual retry function
   - "Retry All" button refreshes entire dashboard
   - OnRefresh handler in PageHeader triggers full reload

Error Messages:
- API errors: Shows error message from server response
- Network errors: Shows generic "Failed to load" message
- Missing data: Shows "No [data type] available yet"

================================================================================
PERFORMANCE CONSIDERATIONS
================================================================================

Optimization Techniques Used:

1. useMemo Hooks
   - testResultsByDay: Recalculates only when testLogs change
   - stageDistribution: Recalculates only when products change
   - symptomInsights: Recalculates only when testLogs change

2. Lazy Calculation
   - Only calculates data for visible chart
   - Symptom keywords extracted on-demand

3. Parallel Data Loading
   - Uses Promise.all() to load all data simultaneously
   - Reduces total load time vs sequential requests

4. Pagination
   - Requests limit of 1000 items (practical limit for dashboard)
   - Avoids massive data transfers

5. Number Formatting
   - Formats large numbers using Intl.NumberFormat
   - Improves readability (1,200 vs 1200)

================================================================================
REFRESH & UPDATE FREQUENCY
================================================================================

Data Refresh:
- Manual refresh via PageHeader button
- Full reload on component mount (useEffect with empty dependency)
- No automatic refresh (manual trigger only)

Real-Time Updates:
- Not real-time; shows snapshot at load time
- Users must manually refresh to see latest data

Caching:
- No caching implemented
- Each refresh fetches fresh data from API

LIMITATIONS & FUTURE IMPROVEMENTS

CONCLUSION
================================================================================

The Dashboard is a comprehensive analytics hub providing real-time insights into

product quality and manufacturing processes.

The modular tab structure allows different user roles to focus on relevant data,
while KPI cards provide at-a-glance status for quick decision-making. Ongoing
data collection and analytics drive continuous improvement in manufacturing
quality and efficiency.
